# AI AppSec Index: A Multi-Dimensional Open Dataset for AI-Augmented Application Security Research

## Authors

Alpha One Index Contributors

## Abstract

The rapid adoption of AI-powered code generation tools has introduced systemic security vulnerabilities into software development workflows. Despite growing awareness, no comprehensive, structured, and openly-accessible dataset exists that spans the full spectrum of AI-augmented application security domains. We introduce the **AI AppSec Index**, an open-source multi-dimensional dataset comprising five complementary components: (1) an AI Remediation Quality Leaderboard benchmarking 30+ tools, (2) an ASPM Capability Matrix evaluating 15+ platforms, (3) an AI-Code Vulnerability Tracker with 48 real CVE mappings across 13 CWE categories, (4) an EU CRA Compliance Map tracking regulatory readiness of 12+ vendors, and (5) a False Positive Benchmark dataset from 20+ SAST/DAST tools. All data is provided in dual JSON/CSV formats with Croissant ML metadata, schema.org structured data, and automated weekly updates via GitHub Actions. To our knowledge, this is the first open dataset that unifies AI remediation benchmarking, vulnerability tracking with real CVE mappings, compliance mapping, and false positive analysis in a single, machine-readable repository.

## 1. Introduction

AI code generation tools such as GitHub Copilot, ChatGPT, Cursor, and Amazon CodeWhisperer have fundamentally changed software development. However, research consistently shows these tools introduce security vulnerabilities at significant rates. Pearce et al. (IEEE S&P 2022) found that GitHub Copilot generates vulnerable code in approximately 40% of security-relevant scenarios. Sandoval et al. (USENIX Security 2023) demonstrated that developers using AI assistants produce code with comparable security to those without, despite writing it faster.

The application security landscape lacks a unified, structured reference that combines:
- **Remediation quality benchmarks** across AI-powered security tools
- **ASPM platform capabilities** in a comparable matrix format
- **Real CVE mappings** to AI-generated vulnerability patterns
- **Regulatory compliance tracking** (EU CRA, NIS2)
- **False positive rates** across commercial and open-source scanners

Existing resources are fragmented: awesome-appsec (6.8k stars) provides learning resources but no structured data; VADER offers vulnerability benchmarks but only for LLM evaluation; SecCodeBench focuses on Java code generation security. None provide the multi-dimensional coverage needed for comprehensive AI AppSec research.

## 2. Dataset Description

### 2.1 AI Remediation Quality Leaderboard

Benchmarks 30+ AI-powered security remediation tools across metrics including fix accuracy, fix completeness, false fix rate, mean time to remediate, and language coverage. Tools evaluated include Snyk DeepCode AI, GitHub Advanced Security, Checkmarx AI, Semgrep AI, and others.

### 2.2 ASPM Capability Matrix

Evaluates 15+ Application Security Posture Management platforms across 20+ capability dimensions including vulnerability correlation, SBOM generation, CI/CD integration depth, runtime protection, and developer experience scoring.

### 2.3 AI-Code Vulnerability Tracker (v2.0)

Tracks 13 CWE categories with real-world CVE mappings:
- **5 AI-tool-specific CVEs**: CVE-2025-53773 (GitHub Copilot RCE), CVE-2025-32711 (Microsoft Copilot EchoLeak), CVE-2025-59944 (Cursor bypass), CVE-2025-54136 (Cursor MCP RCE), CVE-2025-34291 (Langflow RCE)
- **43 related CVEs** mapped to vulnerability patterns commonly generated by AI tools
- Prevalence scoring based on independent security evaluations
- Affected generator tracking across 7+ AI coding assistants

### 2.4 EU CRA Compliance Map

Tracks compliance readiness of 12+ security vendors against EU Cyber Resilience Act requirements, including vulnerability disclosure obligations, SBOM mandates, and incident reporting timelines.

### 2.5 False Positive Benchmark

Comparative false positive rates from 20+ SAST/DAST tools across multiple languages and frameworks, enabling researchers to quantify scanner accuracy and noise reduction capabilities.

## 3. Methodology

Data is collected through:
1. **Primary research**: Direct testing and evaluation of security tools
2. **Academic sources**: IEEE S&P, USENIX Security, ACM CCS, MSR proceedings
3. **CVE databases**: NVD (nvd.nist.gov), MITRE CWE
4. **Vendor documentation**: Security advisories, product specifications
5. **Community contributions**: Verified through peer review process

All data undergoes validation via automated JSON schema checks in our CI/CD pipeline. Updates are published weekly via GitHub Actions.

## 6. Limitations and Future Work

- **Data currency**: Weekly updates may lag behind zero-day disclosures
- **Coverage**: Currently focused on major commercial and open-source tools; niche tools may be underrepresented
- **Validation**: Community contributions require manual review; automated validation covers schema compliance only

Future work includes:
- Kaggle dataset publication for broader ML community access
- Integration with HuggingFace Datasets
- Automated CVE enrichment via NVD API
- Interactive dashboard with filtering and comparison tools
- Expansion to cover AI agent security (MCP, tool-use vulnerabilities)
